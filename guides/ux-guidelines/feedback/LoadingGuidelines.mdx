# Loading component

Loading communicates that work is in progress and results are not yet ready.

## Quick example

<UxExampleProvider>
  <MiniLoadingExample />
</UxExampleProvider>

## Structure

Use loading indicators in the same region where content will appear.

For pane-based screens, use pane-level loading states instead of blocking unrelated areas.

## Behavior

Show loading indicators as soon as a request starts and remove them when data is rendered or an error state replaces loading.

For long operations, keep users oriented with stable layout and avoid content jumping.
