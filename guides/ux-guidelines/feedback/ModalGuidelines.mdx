# Modal component

A modal is a container triggered by a user's action to provide critical information, ask users to make a decision, or support adding content.

## Structure

A modal is a non-dismissible window with a header, content and, at least, one button to close the modal.

The title should be a short, descriptive and clear statement.

The content gives users information about the action they performed and what will happen if they proceed.

Key information should be bold.

Give a solution to users if there is an issue.

Content should be short. To add information while avoiding scrolling, a More info button can be added.

Since modals are not dismissible, they should always have at least one primary button in the bottom right to close the modal, usually labelled Close. If more buttons are needed, they should be in default style with the cancel action on the left.

Labels should be short but descriptive.

Modals should appear at the upper half of the screen and should not take more than 50% of the screen.

## Behavior

When a modal appears, it interrupts a user's current task to catch full attention.

Users consciously dismiss a modal by clicking close or cancel, or by choosing another action.

## Style

Modals can have four states: default, error, success and danger.
