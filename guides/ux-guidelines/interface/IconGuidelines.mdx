# Icon component

Icon communicates state, action intent, or object type in a compact visual form.

## Quick example

<UxExampleProvider>
  <MiniIconExample />
</UxExampleProvider>

## Structure

Use icons that match established meanings so users can recognize actions quickly.

When an icon carries critical meaning, pair it with a visible text label.

## Behavior

Interactive icons should have a clear hit area and keyboard focus treatment.

Status icons should remain consistent across views so severity is predictable.
