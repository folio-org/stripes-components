# Card context

Cards group related content into a bounded visual container.

Guidance here is based on local component usage patterns.

<UxExampleProvider>
  <MiniCardExample />
</UxExampleProvider>

## Structure and style

Cards typically include a header and body region.

Card style and border options can express state (default, positive, negative) while keeping content readable.

## Usage guidance

Use cards for grouped previews, repeating object forms, or linked resource summaries.

Avoid nesting too many levels of cards within cards.
