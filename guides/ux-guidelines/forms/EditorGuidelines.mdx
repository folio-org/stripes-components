# Editor component

Editor supports rich-text authoring when plain text fields do not provide enough structure.

<UxExampleProvider>
  <MiniEditorExample />

## Structure

Use editor fields only where formatting meaningfully improves comprehension or workflow output.

Keep labels and helper text explicit so users know what content and formatting are expected.

## Behavior

Treat editor value as controlled form state and preserve content during validation cycles.

Keyboard users should be able to enter, navigate, and leave the editor without focus traps.
