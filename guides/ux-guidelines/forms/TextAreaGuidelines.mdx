# Text area component

A text area allows users to enter, read and edit text.

## Quick example

<UxExampleProvider>
  <MiniTextAreaExample />
</UxExampleProvider>

## Structure

The text area is a multiline text field.

Generally, it has a label indicating the type of input associated with the field and an area to enter text.

Placeholders should be avoided.

If the field is empty, a null icon can be displayed as a visual indicator.

## Behavior

When users click on a text area, the field should become active.

If the entered text is longer than the field, the content should remain readable and editable.

## Style

Since the text area is a clickable element, it has multiple states.

Each state is associated with a different color, providing visual feedback to users.

Text areas are shown as rectangles with a 1px outline.
