# Segmented control context

Segmented control can be used for filters, tabs, and sub-navigation states within a module view.

Use this guidance where segmented behavior is implemented via higher-level shell or app-specific components.

## Structure

Segmented controls present a small set of mutually exclusive options as adjacent buttons.

One option should be active by default and visibly primary.

## Behavior

Place segmented controls directly above the content region they control.

The visible content should always match the active segment.

Segmented controls should be used only when options are few, stable, and immediately comparable.
