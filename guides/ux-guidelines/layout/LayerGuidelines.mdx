# Layer integration

Layer opens a pane-based full-module overlay for focused tasks such as create, edit, and detail flows.

Use Layer when workflows need a temporary top-level task surface without leaving the current app context.

</UxExampleProvider>

## Structure

Render Layer inside your module Paneset and control visibility through app state.

Within Layer, render a Paneset marked as root so layered panes are managed independently.

Provide a clear, dismissible pane header in the layered content.

## Behavior

Layer behaves like a dialog-level overlay and should be opened only for intentional task transitions.

Set a contentLabel and manage focus on open and close for accessibility.

When Layer closes, return focus to the trigger element when practical.
