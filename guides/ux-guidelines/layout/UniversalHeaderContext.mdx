# Universal header context

The universal header is a platform-level shell component used to help users navigate across FOLIO apps.

The universal header is not controlled at the individual module level, so local apps should not attempt to redesign or override its core behavior.

## What it includes

At a high level, the universal header includes current-app context, app launch areas, global settings actions, and profile access.

Its responsive behavior changes available space and progressively collapses controls as width decreases.

## Integration guidance

Design and prioritize module layouts with the assumption that the universal header is always present above pane content.

When naming app areas and actions, keep labels concise so they remain readable when shell space is constrained.
