# Layout types context

FOLIO app layout is built around a universal header and pane-based content regions.

## Core layout approaches

Search and filter layout: A search/filter pane works with result and record panes so users can refine queries and inspect records quickly.

Hierarchical layout: Parent-child panes represent relationships and support progressive drill-down.

Custom layout: When neither pattern fits, a module can compose a custom pane arrangement while preserving pane and accessibility conventions.

## Integration guidance

Use Paneset to manage pane composition, resize behavior, and state-driven width changes.

Use Layer for temporary top-level flows that should open above existing panes.

Treat Universal Header as shell-owned context that remains above all module content.
