# Pane header component

The pane header helps users identify pane purpose, relationships, and available actions.

## Quick example

<UxExampleProvider>
  <MiniPaneHeaderExample />
</UxExampleProvider>

## Structure

A pane header typically includes a pane title, optional subtitle, and controls.

The action menu should contain actions related to pane content.

Icon buttons should be reserved for pane-level controls such as close, tags, or toggle behaviors.

## Choose the right title format

For result panes, title content should include the record type and optionally a count.

For create or edit states, include concise context that helps users understand whether they are creating, viewing, or editing.

For preview panes, include key identifying characteristics in a scannable format.

## Behavior

Pane header titles should clearly reflect the pane purpose and relationship to adjacent panes.

When users activate the close icon, the pane should collapse and remaining panes should resize.
