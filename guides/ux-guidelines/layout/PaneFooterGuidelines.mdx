# Pane footer component

Pane footer presents persistent actions at the bottom of pane-based forms.

## Quick example

<UxExampleProvider>
  <MiniPaneFooterExample />
</UxExampleProvider>

## Structure

Use start and end action areas to separate secondary actions from primary commit actions.

Keep footer actions stable while pane content scrolls.

## Behavior

Primary actions should be visually emphasized and remain available as users complete long forms.

Footer button order should stay consistent across create and edit workflows.
