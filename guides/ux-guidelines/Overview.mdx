# UX Guidelines

This section migrates guidance from an archived design library into Storybook.

## Content model

- UX Guidelines pages: usage rationale, workflow guidance, accessibility expectations, and examples.
- Component README pages: API-level and implementation-oriented developer documentation.
- Related links connect both surfaces to avoid duplicated content.

## Pilot status

The current migration wave includes 69 drafted pages across seven topics.

- Forms: Text field, Text area, Select, Datepicker, Timepicker, Checkbox, Radio button, Search field, Autosuggest, Repeatable field, Label, Selection, Form field, Advanced search, Editor, Validation and errors.
- Actions: Button, Icon button, Button group, Dropdown button, Export CSV, Confirmation language.
- Feedback: Callout, Message banner, Empty message, Loading, Error boundary, Modal, Modal footer, Error modal, Conflict detection banner, Confirmation modal pattern.
- Navigation: Accordion, Dropdown, Info popover, Filter pane search, Segmented control (context), Nav list, Menu section, Text link, Focus link, Filter groups.
- Data display: List, Multi-column list, Headline, Key value, No value, Highlighter, Record metadata, Accordion core info (context).
- Layout: Layout types (context), Pane, Pane header, Pane menu, Pane footer, Pane subheader, Pane close link, Paneset, Layer, Universal header (context).
- Interface: Avatar, Badge, Icon, Keyboard shortcuts modal, Hot keys, Popover, Popper, Card (context), Tooltip (context).

## Implementation notes

- Prefer live component renderings over static screenshots.
- Use temporary static assets only when interaction fidelity cannot be reproduced yet.
- Track migration state in [migration-manifest.json](./migration-manifest.json).
