# List component

A list allows users to quickly scan through data vertically.

## Quick example

<UxExampleProvider>
  <MiniListExample />
</UxExampleProvider>

## Structure

A list displays a set of items vertically with or without bullet points.

To display more than one list, headlines can be used.

If one list uses a headline, related lists in the same section should also use headlines.

## Behavior

By default, list items are sorted alphabetically.

If no list items are available, display the message `No items to show`.

When users select an item in a selectable list context, the background color should change to indicate selection.
