# Multi-column list component

A multi-column list (MCL) allows users to quickly scan through data both vertically and horizontally.

## Table guidance in FOLIO

When a table-like layout is needed in FOLIO, Multi-column list is the primary pattern.

Use MCL for dense records where users need sortable columns, scanning across fields, and responsive fallback behavior.

## Structure

Typically, an MCL is composed of column headers and row tiles with inline information.

MCL should optimize reading comprehension by alternating row backgrounds and using a fleuron to mark the end of the list.

If needed, checkboxes can be added at the left to allow users to select list items.

## Behavior

When MCL is used in a result pane, selecting an item should open the record pane.

To indicate selection, the selected row should have a stronger background and outline treatment.

When used in edit contexts, quick add/delete controls can appear at the end of rows.

When panes resize, MCL can be scrolled horizontally and may adapt to card-like representations depending on layout.

## Recipes

Use recipe-style controls for customizing visible columns and for selecting and acting on multiple items.
