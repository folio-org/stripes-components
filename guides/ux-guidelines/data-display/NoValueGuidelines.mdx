# No value component

No value shows a consistent placeholder when record data is empty or unavailable.

## Quick example

<UxExampleProvider>
  <MiniNoValueExample />
</UxExampleProvider>

## Usage guidance

Use a single no-value treatment across metadata, summaries, and detail fields to keep scanning predictable.

Prefer explicit no-value placeholders over blank space so missing data is distinguishable from rendering gaps.

## Accessibility

No value should expose screen-reader text that communicates the absence of data.

Do not rely on punctuation-only placeholders without accessible text equivalents.
