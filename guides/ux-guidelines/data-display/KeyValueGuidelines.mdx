# Key value component

Key value structures read-only record information using a label-value pair.

## Quick example

<UxExampleProvider>
  <MiniKeyValueExample />
</UxExampleProvider>

## Structure

A key value contains a key (label) and a value.

The key identifies the data point. The value holds the record information associated with that key.

## Behavior

In read-only display, key and value should be easy to scan and grouped consistently within record sections.

In edit mode, key-value fields should map directly to field labels and input values.

## Style

Keys should be visually lighter than values so values remain the primary content focus.

Keys should appear above values to preserve consistent scanning order.
